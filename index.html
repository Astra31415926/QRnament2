ТЗ для GLM-5: Термінове виправлення звуку Проблема: Поточний звук сканування звучить як безперервна тривожна сирена. Потрібно замінити його на елегантний футуристичний звук із версії 4.5. Інструкції для виправлення: Обмеження тривалості: Звук сканування playScan() НЕ повинен бути нескінченним. Він має тривати рівно 2.5 секунди (час анімації лінії). Налаштування частоти (Pitch): Початкова частота: 400 Hz (м'який тон). Кінцева частота: 1000 Hz (високий технологічний тон). Використовуй exponentialRampToValueAtTime, щоб частота плавно зростала, створюючи ефект "сканування". Гучність (Gain): Встанови м'яку гучність (0.1 або 0.05). Додай Fade-out (плавне зникнення) в останні 0.5 секунди, щоб звук не обривався різко. Тип хвилі: Використовуй тільки sine (синусоїда), вона звучить чисто і приємно. Заборони використання sawtooth або square для режиму сканування, бо вони створюють дратівливий гул. Код для впровадження (коротка логіка): osc.type = 'sine'; osc.frequency.exponentialRampToValueAtTime(1000, now + 2.5); gain.gain.linearRampToValueAtTime(0, now + 2.5);
